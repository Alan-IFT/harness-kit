## Self-template consistency (red lines)

This repo dogfoods two layers of consistency:

### Layer 1: templates ↔ this repo

1. **The source of truth for the 7 agents and `harness-sync` scripts is `skills/harness-init/templates/common/`.**
2. The root `.harness/agents/` and `scripts/harness-sync.{ps1,sh}` are **byte-identical copies**.
3. If you change one, run `scripts/sync-self.{ps1,sh}` before commit.
4. `verify_all` step `E.1` checks this and FAILs on drift.

### Layer 2: .harness ↔ .claude (the binding)

5. **The source of truth for project-level Claude Code assets is `.harness/`** (agents, rules, skills).
6. `.claude/agents/` and `.claude/skills/` are **regenerated** from `.harness/agents/` and `.harness/skills/` via `scripts/harness-sync.{ps1,sh}` (byte-identical copy).
7. **Never hand-edit** files under `.claude/agents/` or `.claude/skills/`. Edit `.harness/` and run sync.
8. `CLAUDE.md` is **not** regenerated by harness-sync since v0.10 — it's a ~15-line static stub pointing at `AI-GUIDE.md`. `AI-GUIDE.md` indexes `.harness/rules/*.md` by reference; AI tools lazy-load fragments on demand.
9. `verify_all` step `E.2` checks `.harness/{agents,skills} ↔ .claude/{agents,skills}` (byte-identity). Step `E.4b` checks `AI-GUIDE.md ↔ .harness/rules/` (bidirectional index). Both FAIL on drift.

## Template integrity

9. **Every `.tmpl` file** must have its placeholders documented in `skills/harness-init/SKILL.md`.
10. Don't introduce a placeholder that the SKILL doesn't substitute.
11. Test the init flow after any template change via `scripts/test-init.{ps1,sh}`.
