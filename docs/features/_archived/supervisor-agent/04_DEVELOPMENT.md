## Insight to surface (optional)

In bash, naming a `while read` loop variable identically to an outer array variable silently clobbers the array via scalar‚Üíarray coercion when later `array+=()` runs; the symptom is a summary counter that's off by 1 (or worse) with no error and no obvious diagnostic in normal output. Detection: `printf '%s\n' "${array[@]}"` shows an unexpected leading empty line. Fix: never name loop variables after globals; defensively use `array=()` over `declare -a array` to keep the failure mode visible under `set -u`. ¬∑ evidence: `scripts/verify_all.sh:450` pre-fix; commit (this delivery)

### Insight to surface

Same insight class as v0.15.0's `-contains` rollback (T-001) and T-002's `-notin` rollback (`insight-index L20`) ‚Äî PowerShell's default operators are case-insensitive, and the bash twin's `=~` / `[[ ]]` is case-sensitive. Every cross-shell symmetry assertion that involves a fixed-case schema needs a `-cmatch` / `-cnotin` / `-ccontains` on the PS side. The existing L20 insight already covers this with both `-cnotin` and the implied general rule; no new insight is needed for v0.17.1 archive ‚Äî this is a recurrence of an already-documented project truth that QA caught via adversarial probe. The win is that the QA adversarial-probe pattern (insight-index encouragement to write negative-fixture reproducers in stage 6) successfully detected the latent regression that all earlier stages missed.
